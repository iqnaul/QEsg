<h2 class="western"><a name="__RefHeading___Toc3061_1012792670"></a> 4.1 DEFINITION</h2>
<p class="western">&nbsp;</p>
<p class="western">The network structure complies with the following principles:</p>
<ul>
<li class="western">The plugin considers the network formed by one or more collectors pipe.</li>
<li class="western">The main collector pipe has the final manhole of the network.</li>
<li class="western">The collector pipes are divided into one or more reaches.</li>
</ul>
<p class="western">&nbsp;</p>
<p class="western">Other basic details are given in the following application example.</p>
<p class="western">&nbsp;</p>

<h2 class="western"><a name="__RefHeading___Toc3065_1012792670"></a> 4.3 APPLICATION EXAMPLE</h2>
<p class="western" style="line-height: 100%;">&nbsp;</p>
<ul>
<li class="western">Set the project to the UTM coordinates, according to the local meridian range.</li>
<li>Load a vector file (shape) with the sewage system. Alternatively you can create a line shapefile and draw the sewer network using the features of QGIS, in this case follow the recommendations below to create the file:
<ul>
<li class="western">Save the file <span style="background: transparent;">(ex.: </span><strong><span style="background: transparent;">Pipes</span></strong><span style="background: transparent;">) </span>using a Coordinate Reference System (CRS) in UTM (Universal Transverse de Mercator) projection type for the local meridian strip.</li>
<li class="western">Always draw the reaches <span id="result_box" class="short_text" lang="en"><span class="">attempting</span></span> to the flow direction (from upstream to downstream).</li>
<li class="western">It is not necessary to create any field when creating the shapefile <strong>Pipes</strong>.</li>
</ul>
</li>
<li>Save the Project. <span id="result_box" class="" lang="en"><span class="">At this point the project window will look like <strong>Figure 5</strong>.</span></span></li>
<li><span class="" lang="en"><span class="">Open <strong>Pipes </strong>attribute table where there will be one line for each collector and only one field with null values (see <strong>Figure 5).</strong>&nbsp;</span></span></li>
</ul>
